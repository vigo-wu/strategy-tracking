配合主 Skill **D. 回测** / **E. 策略变更**。以主题 `model.md` 为准；本文件固化仓库内已跑通的流程。

## 1. list.md 股票池

表格格式（主题目录或 `--list` 路径）：

```markdown
| 序号 | 股票代码 | 股票名称 |
| :--- | :--- | :--- |
| 01 | 601919 | 中远海控 |
```

脚本解析：正则匹配 `| 数字 | 六位代码 | 名称 |`，生成 `code6` + baostock `sh./sz.`。

```bash
python scripts/run_backtest.py --universe list
python scripts/run_backtest.py --universe list --list ../list.md
```

输出文件名：`backtest_*_list_YYYYMMDD_YYYYMMDD.*`（`--universe list` 时加 `list_` 前缀）。

## 2. 定额资金组合（示例：总盘 20 万 / 单笔 5 万）

```text
TOTAL_CAPITAL = 200_000
PER_TRADE_BUDGET = 50_000
MAX_SLOTS = TOTAL_CAPITAL // PER_TRADE_BUDGET   # 4
```

### 配资流程

1. 各标的按 `model.md` 规则生成**信号候选**（含模拟出场）  
2. 按 `(entry_date, code)` 排序  
3. 入场前先结算 `exit_date <= 当日` 的持仓，释放现金  
4. 整手：`shares = floor(budget / (price * 100)) * 100`；不足 1 手 → 跳过  
5. 已持仓数 ≥ MAX_SLOTS → 跳过（原因：`满仓位(已4仓)`）  
6. `cost > cash` → 跳过（原因：`现金不足`）  
7. 成交写入 trades；跳过写入 skipped  

R2 减半：回测可将半仓出场与余仓出场**加权合并为一笔** `ret_pct` / `pnl`，报告中标注 `R2` 是否触发。

## 3. 回测报告必须章节

`backtest_report_*.md` 建议结构：

1. **标题**（含策略版本、list 字样）  
2. **摘要子弹列表**（区间、池、资金、信号/成交/跳过、期末权益、胜率、出场分布…）  
3. **假设**  
4. **零信号标的**  
5. **分票盈亏 Top5 / Bottom5**  
6. **信号交易明细（按入场日 · 已成交）** — 用户常要求「列在回测报告中」  
7. **跳过信号**（若有）  

### 交易明细列（定额模式）

`# | 代码 | 名称 | 入场日 | 出场日 | 买入 | 有效卖出 | 股数 | 成本 | 盈亏 | 收益% | R2 | 出场`

代码列须 **六位补零**（如 `001872`，勿写成 `1872`）。

## 4. 策略变更后重跑

变更出场/入场后：

1. 改 `model.md` + 脚本  
2. `python scripts/run_backtest.py --universe list`  
3. 覆盖/更新同 stamp 的 report/trades/summary  
4. 更新当日日志「历史回测摘要」与 `log/README` 索引一行  

可在日志中简要对比旧版关键指标（胜率、总收益率、成交笔数），勿删用户未要求删除的旧产物。
